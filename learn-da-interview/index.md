# DA 面试

学完了技术和商业分析的知识，最终的目标是**拿到 Offer**。数据分析师面试有其独特的考核体系——不仅考你会不会写 SQL，还考你能不能做 Case Study、讲清楚业务逻辑、展示分析思维。

本模块系统覆盖数据分析师面试的各个环节，从面试准备到各类题型的实战练习。

## 面试类型总览

数据分析师面试通常包含以下环节：

| 环节 | 考察内容 | 占比（参考） | 本模块对应文章 |
|------|---------|------------|--------------|
| 简历筛选 & HR 面 | 背景匹配度、沟通能力 | — | 面试通关指南 |
| SQL 笔试/面试 | SQL 查询能力 | 30-40% | SQL 简单/中等/困难题 |
| Case Study | 商业分析思维 | 20-30% | Case 框架 + 产品指标 Case |
| 行为面试（BQ） | 过往经历、团队协作 | 15-20% | BA/DA 高频面试题 |
| 技术深度面 | 统计、A/B 测试、工具使用 | 10-20% | BA/DA 岗位解析 |

## 学习建议

> **前置知识**：SQL 模块是必须的前置模块。商业分析模块和数据思维模块对 Case Study 环节帮助极大。

- SQL 面试题按难度分为三级，建议从简单题开始，确保基础题不丢分
- Case Study 的关键不是"答案正确"，而是"思路清晰、逻辑自洽"
- BA 和 DA 虽然都叫"分析师"，但面试侧重点不同——BA 更偏业务理解，DA 更偏技术能力
- 每篇文章的面试题都附有详细解析，建议先自己尝试回答，再看参考答案
